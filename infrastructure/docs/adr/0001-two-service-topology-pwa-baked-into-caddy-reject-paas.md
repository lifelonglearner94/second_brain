# Two-service Compose topology with PWA baked into Caddy, rejecting PaaS

Single-user, Personal-Scale: a custom Caddy image (TLS termination, `file_server` for the built PWA, `/api` reverse-proxy) and the Rust backend — two long-running services, no frontend container. The PWA Bundle is baked into the Caddy image at build time (not a shared volume) to avoid volume permission/ownership pain and keep the Edge stateless and disposable. PaaS options (Coolify/Dokploy) are explicitly rejected: their control plane (Traefik + PostgreSQL + Redis + workers) steals RAM from sqlite-vec and the backend on an 8GB VPS, introducing fleet-management overhead for a system with one user. Raw `docker-compose.yml` + a ~5-line Caddyfile + a GitHub Actions SSH-deploy script give absolute control with zero idle overhead.
