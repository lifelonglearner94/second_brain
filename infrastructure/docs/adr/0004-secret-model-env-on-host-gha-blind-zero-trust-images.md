# Secret model: .env on host, GHA-blind, Zero-Trust Images

Runtime secrets (Deepgram, Cohere, Gemini API keys; R2 credentials for Litestream) live in a single `.env` file on the VPS host, injected into the backend container via `env_file:`. The file is placed manually once over SSH and never touched by CI - GHA is completely blind to runtime secrets and only ships code. This deliberately accepts the at-rest risk: if an attacker has root on the VPS, they already have the Brain File (the entire brain), so the `.env` file is not additional threat surface worth mitigating. Docker Compose secrets and secrets managers (Doppler, Infisical, Vault) are rejected - the former adds syntactic overhead without mitigating root compromise; the latter violates the Personal-Scale mandate that rejected Coolify. The complementary constraint is Zero-Trust Images: no secret ever touches a Dockerfile `ENV` or `COPY` directive, so images remain public-safe artifacts - if a GHCR image leaked to the public internet, only compiled code would be exposed, zero credentials. Rejected secret-delivery flows: GHA writes `.env` from GHA secrets on every deploy (a compromised GitHub account would instantly leak the entire infrastructure access - GHA is a CI server, not a secret manager).
