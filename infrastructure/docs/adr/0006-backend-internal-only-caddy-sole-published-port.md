# Backend internal-only: Caddy is the sole published port

The backend service in Compose has no `ports:` mapping to the host - it is reachable only on the internal Docker network (`app_network`), and Caddy routes to it as `http://backend:8080`. The only published ports are `:80` (ACME HTTP-01 challenges) and `:443` (TLS) on Caddy. Exposing the backend via `ports: "8080:8080"` would bind it to `0.0.0.0`, bypassing Caddy entirely - stripping TLS, defeating edge-level rate limiting, and exposing the raw Axum API to internet port scanners. Even with Passkey authentication (not a catastrophic auth breach), this is an unacceptable failure of defense-in-depth. The "missing" `ports:` block on the backend is deliberate, not an oversight - a future maintainer running `docker compose ps` who sees no exposed ports and "fixes" it by adding them reintroduces the bypass. The backend uses `expose:` (documentation-only, does not publish to host) rather than `ports:` to make the intent explicit.
