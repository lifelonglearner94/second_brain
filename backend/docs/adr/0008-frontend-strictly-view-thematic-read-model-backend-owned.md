# Frontend is strictly a view; thematic clustering is a backend-owned read model

The first_draft placed graphology (with ForceAtlas2 layout and Louvain community detection) in the frontend as "the data model." We split this: the frontend renders, the backend owns all graph computation — including thematic clustering (Louvain), which is a computation over graph topology, not a rendering concern. Clusters are a third read surface — the Thematic Read Model — alongside Retrieval (ADR-0004) and Chat (ADR-0005/0006).

## Considered options

- **Frontend-local clustering** (graphology + Louvain in the browser, as first_draft proposed): rejected — breaks the "frontend is just a view" boundary. The graph topology is backend-owned; shipping it to the frontend to compute locally makes the frontend a co-owner of the model, not a view over it. Layout (ForceAtlas2) is visual; community detection (Louvain) is semantic-adjacent — both are computations over the graph, and the graph lives in the backend.
- **Clusters as first-class entities** (with identity, provenance, accretion — symmetric to Concepts and Edges): rejected — non-deterministic partitions don't have stable identity across sessions. If a cluster deserves persistence, it does not gain identity retroactively; it routes through ADR-0006's governed write-back as a proposed new Concept, and persistence is granted at the concept layer.

## Consequences

- The frontend renders clusters the backend computed; it does not run Louvain itself. graphology in the frontend is a layout/render cache, not a data model (correcting first_draft.md).
- **Closes ADR-0006's open thread** (the "new concept" fan-out): a transient cluster significant enough to persist is the canonical motivation — the cluster does not gain identity; instead, a new Concept is proposed through governed write-back, routed through concept identity (ADR-0001) and edge/type governance (ADR-0002/0003) for its links to cluster members.
- The Thematic Read Model is a third read surface, **layered into Chat** (not parallel): Louvain computes the global macrostructure; the LLM uses the current partition as additional context for local micro-inference. Anything else would exceed the context window — the LLM cannot re-derive the global structure from raw edges within budget.
- **Not layered into Retrieval (ADR-0004).** Retrieval remains strictly seed-then-expand along *typed edges* — cluster membership is never a traversal mechanism. Non-deterministic retrieval undermines trust in RAG and makes debugging impossible; the graph must remain robust infrastructure at the retrieval level. This accepts a blind spot: concepts thematically linked across many braindumps but with no direct typed edge between them are invisible to Retrieval.
- **The blind spot is the growth engine, not a defect.** A gap between thematic density (Louvain sees a cluster) and edge sparsity (no typed edge connects the cluster-mates) is exactly the signal that motivates an ADR-0006 write-back proposal — the LLM, fed the current partition, proposes edges to fill the gaps Louvain found. As the graph densifies over time via endorsed proposals, the blind spot systematically shrinks.
- **Promoted to Hypothesis Engine (ADR-0006 split).** The Thematic Read Model is not just visualization or Chat context — it is the system's Hypothesis Engine, the sole source of Thematic Inference proposals. Thematic Inference is the proposal mode that introduces structure not derivable from the existing graph (vs Structural Inference, which only labels existing edge paths). This makes the Thematic Read Model load-bearing for graph growth, not just for reading.
- **Cluster labels are ephemeral session labels**, not stable IDs. Within a single partition computation, clusters carry throwaway labels ("Group 1 for this session") so the LLM can reference them in its reasoning; these labels die with the partition. The LLM may use clusters as a magnifying glass to find things, but must never cite a cluster as a final source — every claim cites the underlying braindumps and edges (the stable truth), never the ephemeral projection.
